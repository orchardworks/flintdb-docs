## Why FlintDB?

<CardGrid>
  <Card title="JSON-Native" icon="document">
    Store and query JSON documents directly. No schema definitions, no migrations — your data's shape is your schema.
  </Card>
  <Card title="Built-in Analytics" icon="bars">
    avg, stddev, EMA, percentile, correlation — computed inside the engine. No Pandas, no ETL pipeline.
  </Card>
  <Card title="Query is Data" icon="pencil">
    Queries are plain JSON objects, not SQL strings. No parser, no DSL — just objects in, results out.
  </Card>
  <Card title="Vector Search" icon="puzzle">
    Store embeddings alongside your documents. Cosine and L2 nearest-neighbor search built in.
  </Card>
</CardGrid>
